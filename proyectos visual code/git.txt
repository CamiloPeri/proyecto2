--------- tres estados de git-------
committed significa confirmado   // siginifica que los datos
 estan almacenados en una base local 

modified : modificado/ significa que se ha hecho una modificacion 
pero aun no se ha confirmado en la base de datos


staged : preparado // significa que  se ha marcado un archivo  modificado  en una
actual ara que se haga una proxima confirmación


----CONCLUSIÓN----
-  Modificas una serie de archivos en tu directorio de trabajo.
-  Preparas los archivos, añadiendolos a tu área de preparación.
-  Confirmas los cambios, lo que toma los archivos tal y como están en el
 área de preparación, y almacena esas instantáneas de manera permanente en tu directorio de Git.
